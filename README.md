# Трекер привычек план/факт

Личный продукт для учета привычек по месяцам: план, факт, прогноз, риски, цели, заметки, отчеты и локальные напоминания. Приложение заменяет таблицу план/факт и работает как PWA на телефоне.

## Стек

- Next.js App Router, TypeScript, React
- Tailwind CSS и shadcn/ui-style компоненты
- Recharts для графиков
- Supabase Auth и Supabase Database
- React Hook Form и Zod
- date-fns
- Vitest
- PWA: manifest, service worker, offline fallback, install prompt

ИИ API не используются. OpenAI, AI SDK, LangChain и внешние LLM не подключены. Все рекомендации генерируются локальными rule-based функциями.

## Возможности

- месяцы, категории, задачи, веса и календарный план;
- генератор плана: каждый день, будни, выходные, дни недели, конкретные даты, N раз в месяц, вручную;
- создание месяца из шаблона прошлого месяца;
- утверждение плана с защитой от уменьшения;
- быстрый дневной ввод факта по шкале 0–2 с шагом 0.25;
- массовые действия в дневном вводе, копирование вчерашнего факта, дневная заметка, настроение и энергия;
- дашборд с KPI, прогнозом, главным фокусом, рисками и lazy-loaded графиками;
- календарь, аналитика, недельный и месячный отчеты;
- цели с привязкой к задачам;
- заметки с поиском, тегами и пагинацией;
- история месяцев с пагинацией, закрытие и разблокировка месяца;
- проверки качества данных на `/checks`;
- JSON экспорт/импорт с предварительным summary;
- PWA-установка, offline fallback и кеширование основных страниц;
- локальные browser notifications без backend worker.

## Запуск локально

1. Установите зависимости:

```bash
pnpm install
```

2. Создайте `.env.local`:

```bash
cp .env.example .env.local
```

3. Заполните переменные:

```env
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
```

4. Примените SQL миграции из `supabase/migrations` по порядку.

5. Запустите dev-сервер:

```bash
pnpm dev
```

6. Откройте `http://localhost:3000`.

### Запуск с телефона в одной Wi-Fi сети

Для телефона используйте отдельный запуск:

```bash
pnpm dev:lan
```

Или дважды нажмите файл:

```text
Запустить трекер для телефона.cmd
```

Скрипт покажет адрес вида:

```text
http://192.168.x.x:3000/mobile
```

Откройте этот адрес на телефоне. Ноутбук и телефон должны быть подключены к одной Wi-Fi сети. Если страница не открывается, разрешите Node.js в Windows Firewall для частной сети.

`localhost:3000` на телефоне не подходит: для телефона `localhost` означает сам телефон, а не ноутбук.

Для постоянного доступа, когда ноутбук выключен, нужен онлайн-деплой, например Vercel. Локальный Wi-Fi запуск работает только пока ноутбук включен и сервер запущен.

## Supabase

Основные таблицы:

- `profiles`
- `user_preferences`
- `categories`
- `tasks`
- `months`
- `daily_plans`
- `daily_facts`
- `daily_notes`
- `task_planning_rules`
- `goals`
- `goal_tasks`
- `notes`
- `change_logs`

RLS включен на пользовательских таблицах. Пользователь видит и меняет только свои данные. Баллы плана и факта пересчитываются триггерами по весу задачи. Утвержденный план нельзя уменьшить или удалить. Закрытый месяц защищает факты и дневные заметки от случайного редактирования.

Индексы добавлены для частых запросов:

- `daily_plans(month_id, date)`
- `daily_plans(task_id, date)`
- `daily_facts(month_id, date)`
- `daily_facts(task_id, date)`
- `tasks(user_id, category_id)`
- `months(user_id, year, month)`
- `notes(user_id, date)`
- `goals(user_id, status)`

## Данные

Демо-загрузка из интерфейса отключена. Для личного использования создайте свои категории, задачи и месяц на странице `/planner`, затем сгенерируйте или заполните план вручную.

Для переноса данных используйте экспорт и импорт JSON на странице `/settings`.

## Командный учет

Раздел `/team` добавляет командный слой поверх личных планов:

- личные планы, задачи и факты остаются у каждого пользователя отдельно;
- участники команды видят общий прогресс, прогноз, вклад участников и командные риски;
- редактировать чужие планы и факты нельзя;
- приглашение создается ссылкой и принимается на `/team/invite/[token]`.

Для включения команд примените миграцию:

```text
supabase/migrations/202606190001_team_collaboration.sql
```

Миграция добавляет таблицы `teams`, `team_members`, `team_invites` и RLS-политики для безопасного командного просмотра.

## Онлайн-запуск

Для доступа с телефона и для друзей приложение нужно разместить в облаке. Рекомендуемый путь:

1. Залить проект в GitHub.
2. Подключить репозиторий к Vercel.
3. Добавить в Vercel переменные:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. В Supabase Auth добавить домен Vercel в разрешенные URL.
5. Применить все SQL-миграции в Supabase.

После деплоя пользователи заходят по ссылке Vercel, а ноутбук не нужен для работы приложения.

## PWA и уведомления

PWA файлы:

- `public/manifest.json`
- `public/sw.js`
- `public/icons/icon.svg`
- `public/icons/maskable.svg`
- `/offline`

Установка появляется через browser install prompt. Service worker кеширует основные страницы и показывает offline fallback при потере сети.

Локальные уведомления настраиваются в `/settings`:

- включить ежедневное напоминание;
- выбрать время;
- включить предупреждения о рисках;
- тестовое уведомление;
- тема: светлая, темная или системная.

Уведомления работают только после разрешения браузера и без внешнего worker: когда приложение открыто, оно планирует локальные browser notifications.

## Проверки

```bash
pnpm typecheck

pnpm build
```

Unit-тесты покрывают:

- метрики и статусы;
- rule-based рекомендации;
- генератор планов и шаблонов;
- Zod validators.

Playwright e2e не добавлен, потому что проект не настроен под Playwright.

## Деплой

1. Создайте Supabase project.
2. Примените миграции.
3. Укажите env variables на хостинге.
4. Выполните `pnpm build`.
5. Разверните как обычное Next.js приложение.

Для Vercel достаточно подключить репозиторий и добавить:

```env
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
```
